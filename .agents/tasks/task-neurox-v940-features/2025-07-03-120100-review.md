# Phase 4 market microstructure modules (v2)

Six new market microstructure modules integrated into the main trading loop, with fix commit resolving the four primary issues from v1: tick frequency amplifier now scales lot size, MTF confluence multiplier compounds onto it, cumulative delta divergence fires breakout signals and blocks contradicting momentum, and liquidity sweep now respects the spread gate. Version bumped to 9.40/20250703 across all required files. 346 tests pass; 70 are new.

**Watch for:** Delta breakout signals fire without persistence, capacity, hysteresis, or ATR checks — only cooldown, spread, and direction lock — making them the least-gated signal path in the system (confirmed). The spread proxy measures tick-to-tick volatility rather than actual spread, which can suppress trading in ranging mode during normal choppy consolidation (likely). The tick deque capacity (maxlen=50) still constrains delta window effectiveness during institutional spikes (confirmed, pre-existing from v1).

**Verdict**: APPROVED

## High-level view

The v1 review's primary concern — four modules computed but discarded — is resolved. All six modules now influence trading decisions: tick frequency amplifies lot size, MTF confluence further scales it for bar-based signals, cumulative delta fires breakout signals and blocks contradicting momentum in trending mode, spread gates all signal paths (including sweep), liquidity sweep fires instant reversals, and S/R bounce informs mean-reversion direction.

The lot multiplier compounds tick-frequency amplification (1.0–1.5x) with MTF position sizing (0.5–1.5x), producing a maximum 2.25x multiplier clamped by MAX_LOT_SIZE. On tick-only iterations (before bars form), only the tick-frequency component applies. The multiplier resets each iteration, preventing stale state.

Delta divergence acts as both a signal generator (BREAKOUT_UP/DOWN fire trades) and a signal blocker (BEARISH divergence suppresses BUY, BULLISH suppresses SELL in trending mode). The blocking applies only to the bar-momentum path — tick-momentum signals, velocity spikes, sweep signals, and V-reversal patterns are unaffected by divergence warnings.

## Delta breakout as the least-gated signal path

The cumulative delta divergence wiring fires signals directly when `divergence_type` is BREAKOUT_UP or BREAKOUT_DOWN:

```python
if divergence_info["divergence_detected"]:
    div_type = divergence_info["divergence_type"]
    if div_type == "BREAKOUT_UP" and can_trade() and spread_allows_trading \
            and direction_matches_open("BUY"):
        fire_signal(bridge, "BUY", current_price,
                    "DELTA BREAKOUT UP", regime="momentum", model_name="delta_v9")
```

The bar-momentum path requires 7 gates (cooldown, capacity, direction lock, hysteresis, ATR, tick consistency, persistence). The delta breakout path requires 3 (cooldown, spread, direction). The liquidity sweep also uses 3 gates, defensible because sweeps are instant-reversal signals. Delta breakouts fire on price-flat-but-delta-strong conditions — a less urgent pattern that doesn't warrant bypassing capacity checks.

During an institutional tick spike (50+ tps), the tick deque churns rapidly, and each loop iteration could produce a fresh BREAKOUT detection (delta window is 30 ticks = 0.6 seconds at 50 tps). The 5-second cooldown limits firing frequency, but without `has_capacity()` the system could fill to MAX_POSITIONS purely on delta breakouts in 25 seconds.

## Spread proxy suppressing ranging-mode trading

The `compute_spread_from_ticks` function uses `np.median(abs(tick[i] - tick[i-1]))` as the spread estimate. During a fast unidirectional move over 20 ticks, the median diff stays small (~$0.025 for a $0.50 move), so the proxy rarely blocks trending signals.

The risk materializes in ranging mode: during choppy back-and-forth consolidation (common in gold during Asian session), the median diff can exceed the WIDE threshold ($0.15) even though actual spread is normal. The spread gate would suppress mean-reversion trading precisely when that strategy should be most active — the choppy ticks that make the proxy read "wide" are exactly the conditions where mean reversion fires.

## Divergence blocking and persistence interaction

The divergence block applies before the filter chain in trending mode:

```python
elif (divergence_info["divergence_detected"]
      and divergence_info["divergence_type"] == "BEARISH"
      and momentum == "BUY"):
    intel_decision = "FILTERED"
    intel_reason = "DELTA_DIVERGENCE"
```

Because the block short-circuits before `passes_signal_persistence()` is called, a divergence-blocked signal doesn't increment `signal_persistence_count`. If divergence oscillates on/off (one iteration has it, next doesn't), persistence may never accumulate to the required count of 2. Oscillating divergence detection could effectively prevent any bar-momentum signal from firing in trending mode during volatile periods when delta readings fluctuate rapidly.

## Tick deque capacity constraint (carried from v1)

The `_tick_prices` and `_tick_timestamps` deques remain at maxlen=50. At 50+ tps, the delta window (30 ticks) sees 0.6 seconds and the spread window (20 ticks) sees 0.4 seconds — short enough that a single large tick-to-tick jump dominates the calculation. This limits the value of delta divergence detection during institutional spikes where it would be most useful. Pre-existing; not introduced in this PR.
