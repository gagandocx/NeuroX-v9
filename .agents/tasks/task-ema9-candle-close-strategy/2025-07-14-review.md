# EMA 9 candle-close strategy with multi-indicator choppy filter

The trading logic is overhauled in two parts: direction determination switches from EMA 9/15 crossover to price-vs-EMA-9 only (with a $0.80 max-distance gate), and the exit system replaces a 4-tier progressive trailing stop with a candle-close cycle (trade closes when the M1 bar closes, no TP, SL at the last swing high/low). A new choppy-market filter votes across five indicators — ADX, Choppiness Index, Bollinger Band width, ATR ratio, and variance ratio — to block entries in ranging conditions. The EA side is simplified to breakeven-only management ($5 profit moves SL to entry + $1), with Python owning candle-close exits and M1 reversal detection.

**Watch for:** The choppy filter's ATR-ratio and variance-ratio votes are never active in production because `current_atr` and `avg_atr` are never passed from `main.py` (confirmed). The EA sends swing high/low values over the pipe but Python ignores them, recomputing swing levels from tick-collector bars that may not align with the EA's 20-bar lookback (likely). Reversal detection depends on `tick_collector._completed_bars` which starts empty on each Python restart and accumulates slowly — meaning the safety net is unavailable for the first ~10 minutes of operation (confirmed).

## High-level view

The direction model is now trivially simple: price above EMA 9 is bullish, below is bearish. The $0.80 distance cap keeps entries tight to the EMA, which on XAUUSD M1 means roughly 1-2 entries per trending minute candle.

The choppy filter has five uncorrelated indicators with a configurable agreement threshold, but two of the five voters (ATR ratio and variance ratio) are currently unreachable because `main.py` doesn't supply `current_atr` or `avg_atr` to `is_market_choppy()`. The filter operates as a 3-indicator system in practice (ADX, Choppiness Index, BB squeeze), and the default threshold of 2 means any two of those three suffice to block.

The candle-close exit replaces all trailing logic. `CandleCloseManager` runs a 100ms background loop checking three conditions in priority order: (1) high-momentum reversal candle against the trade fires early CLOSE_FULL, (2) minute change fires CLOSE_FULL, (3) $5 profit fires MODIFY_SL for breakeven. MAX_POSITIONS drops from 5 to 1, enforcing a single-position candle-close cycle.

The swing SL is computed in Python from `tick_collector._completed_bars` using a fractal-style detection (bar higher/lower than N neighbors). The EA also computes and transmits its own swing high/low (simple max/min of 20 bars), but Python ignores those values. On startup or reconnect the Python-side buffer is empty, producing fallback SLs at the minimum $0.50 distance — extremely tight for XAUUSD.

The EA's `beLock` is hardcoded at $1.00 rather than reading from a configurable input, while Python has `Config.BREAKEVEN_LOCK_AMOUNT = 1.00`. If the user changes the Python config, the EA won't match.

## Choppy filter: 3 of 5 voters active

The `is_market_choppy()` signature accepts `current_atr`, `avg_atr`, and `variance_ratio` with defaults of 0.0, 0.0, and 1.0 respectively. With those defaults:

- ATR ratio check: `current_atr / avg_atr` would be `0.0 / 0.0` which is guarded by `if current_atr > 0 and avg_atr > 0:` — skipped.
- Variance ratio check: `variance_ratio > 0 and variance_ratio < 0.5` — default is 1.0, so never triggers.

The call site in `main.py` (line 229-234) passes only `adx_value`, `choppiness_index`, `bb_upper`, `bb_lower`, and `current_price`. The EA pipe doesn't transmit raw ATR or average ATR values. To fix this, either extend the pipe format to include these values, or compute them Python-side from the tick collector's completed bars.

The ATR-ratio voter would catch the specific case where the market is directional but on low absolute volatility — a condition the other three don't cover well.

## Dual swing SL: EA pipe vs Python computation

The EA computes swing levels as the simple max high and min low of the last 20 M1 bars:

```c
swing_high = highs[0];
swing_low = lows[0];
for(int i = 1; i < 20; i++) {
    if(highs[i] > swing_high) swing_high = highs[i];
    if(lows[i] < swing_low) swing_low = lows[i];
}
```

These are transmitted in the pipe and parsed by `read_ema_from_ea()` into `ea_swing_high` and `ea_swing_low`, which are then unused — no reference to them appears after the destructuring assignment.

Python's `compute_swing_sl()` uses a fractal detection (bar higher/lower than N neighbors on both sides) with `SWING_SL_LOOKBACK=10` and `swing_width=2`. This is a more sophisticated algorithm but it operates on `tick_collector._completed_bars`, a deque(maxlen=20) that starts empty every time the Python process launches.

The consequence: the first trade after startup uses the fallback SL (`entry_price - $0.50` for BUY, `entry_price + $0.50` for SELL). On XAUUSD, a $0.50 SL is about 5 pips — extremely tight. Any normal M1 noise would trigger it.

Fix: use `ea_swing_high`/`ea_swing_low` from the pipe as a fallback or primary SL source when the Python bar buffer is insufficient. The EA has all 20 bars available from MetaTrader's history immediately.

## Candle-close re-entry timing

After Python fires CLOSE_FULL, the EA needs time to execute the close and decrement the position counter in the pipe file. If the Python loop runs faster than the EA execution, `ea_open_positions >= MAX_POSITIONS` blocks re-entry for one or more loop cycles even though Python has already stopped tracking. This delays re-entry by at most a few hundred milliseconds, meaning the system won't always re-enter on the very next M1 candle as the architecture intends.

## Reversal detection: blind spot in early candle

`_check_reversal()` reads the *current forming bar's* OHLC from the tick collector. If the bar just opened (first few seconds), body and range will be tiny and reversal criteria won't be met. The protection only activates once the bar is well-formed — typically 20-40 seconds into a candle. Extremely fast reversals (first 5 seconds of a new candle) won't be caught by this mechanism; the swing SL is the only backstop.

## Test coverage gaps

Not tested:
- The cold-start fallback path where `completed_bars` is empty and swing SL defaults to $0.50
- ATR ratio and variance ratio voters (no test catches that they're unreachable in production)
- Race condition between `close_fired` check and next-iteration re-entry
- The EA-side breakeven $1 hardcode vs Python `Config.BREAKEVEN_LOCK_AMOUNT` alignment
