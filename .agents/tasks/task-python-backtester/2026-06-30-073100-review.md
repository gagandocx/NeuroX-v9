# Python backtester for NeuroX v9 trading logic

Standalone backtester (`backtest.py`, 736 lines) that replays M1 OHLC CSV data through both the v9.1 (momentum-only) and v9.2 (full-filter) trading logic, producing a side-by-side comparison report. The script delegates signal computation to the existing `momentum.py` / `config.py` modules and adds its own trailing-stop simulation, position lifecycle, and CSV parsing. A `data/README.md` documents the expected MT5 export format.

**Watch for:** The trailing-stop tiers hardcoded in the backtester diverge from the `Config.TRAIL_TIERS` in `config.py` (confirmed). Tick consistency is approximated from bar closes with a lookback of 40 but only ~1-minute resolution, making the 60% threshold a very different gate than it is on real tick data (confirmed). Max-hold is mapped to 2 bars (120s / 60s), which silently becomes incorrect if the user supplies M5 or H1 data (likely). Exit on file-not-found and parse-error uses `sys.exit(0)` instead of a non-zero code (confirmed).

## High-level view

The backtester wires together existing signal-generation code (`compute_momentum`, `detect_regime`, `compute_mean_reversion_signal`) with a new bar-by-bar position-management engine. It does not re-implement the indicators — it imports them — so fidelity to the EA depends entirely on how well the bar-level simulation approximates tick-level behavior.

The trailing-stop system is the most complex new code. It models the 4-tier tightening from the EA's `NeuroX_Position.mqh`, but the dollar thresholds used here (BE=$0.30, T2=$0.60, T3=$1.20, T4=$2.00) do not match the `Config.TRAIL_TIERS` table (which starts at $0.50/$1.00/$2.00/$3.00). One of these is stale; the discrepancy means backtester results won't match either the live EA or the Python config.

Spread handling splits cost evenly: half-spread on entry, half-spread on exit. The EA pays full spread on entry (Ask for BUY, Bid for SELL). Net round-trip cost is identical, but the stored `entry_price` is mid+half-spread rather than the actual Ask/Bid, which shifts the profit thresholds for trailing-stop tier activation by half a spread.

The multi-position strong-momentum gate compares raw price movement against a fixed $0.45 threshold (`TICK_MOMENTUM_THRESHOLD * STRONG_MOMENTUM_MULT`). Unlike `compute_momentum` itself, this gate does not scale with ATR, so it's permissive in volatile sessions and restrictive in calm ones.

4. **Exit codes are always 0** — `sys.exit(0)` on error paths (file not found, parse failure) prevents scripted pipelines from detecting failures. Use `sys.exit(1)`.
5. **Cooldown is 1 bar, not 5 seconds** — The cooldown check (`bar_idx <= self.last_trade_bar`) blocks only for the same bar. With M1 data one bar already exceeds the 5s cooldown, but sub-minute data would break this.
6. **Volume column dropped changes momentum signal** — The backtester drops `Volume` before calling `compute_momentum` to avoid volume-weighted mode, meaning backtester signals diverge from live. Document the trade-off.
7. **No test coverage** — No unit or integration tests for trailing stops, CSV parser, or position lifecycle. Edge cases (empty CSV, single bar, spread=0) are unguarded.

## Trailing-stop tier divergence from Config

The backtester defines its own constants at module level:

```python
BE_PROFIT_THRESHOLD = 0.30
T2_PROFIT_THRESHOLD = 0.60
T3_PROFIT_THRESHOLD = 1.20
T4_PROFIT_THRESHOLD = 2.00
```

Meanwhile `Config.TRAIL_TIERS` says:

```python
TRAIL_TIERS = [
    (0.50, 0.00),   # After $0.50 profit: trail at breakeven
    (1.00, 0.50),   # After $1.00 profit: lock $0.50
    (2.00, 1.50),   # After $2.00 profit: lock $1.50
    (3.00, 2.50),   # After $3.00 profit: lock $2.50
]
```

The user instruction specifies BE=$0.30, T2=$0.60, T3=$1.20, T4=$2.00, so the backtester matches the requirement. But `Config.TRAIL_TIERS` is now out of sync — a future contributor reading the config will get wrong numbers. Either update the config or add a comment in `backtest.py` explaining the authoritative source is the EA header file.

## Max-hold time assumes bar period = 60 seconds

```python
bars_held = bar_idx - pos.entry_bar
if bars_held >= 2:  # MAX_HOLD_SECONDS=120 / 60s per bar = 2 bars
```

The CSV has Date/Time columns that are discarded during parsing. Timestamp-based hold enforcement would handle arbitrary timeframes correctly.

## Tick-consistency approximation fidelity

On live data, `TICK_CONSISTENCY_LOOKBACK = 40` inspects 40 individual ticks arriving in sub-second bursts. In the backtester, `compute_tick_consistency` inspects the last 40 M1 bar closes — a 40-minute window. The distribution of up/down moves across 40 minutes behaves very differently from 40 consecutive ticks. The filter will pass or fail at different rates than the live system, which undermines the side-by-side comparison goal. This limitation should be documented prominently in the output report so users don't mistake backtester results for live-equivalent performance.

## Spread asymmetry in trailing-stop profit calculation

```python
# In compute_trailing_sl:
if position.direction == "BUY":
    profit = current_price - position.entry_price
```

`current_price` is bar close (the mid-ish price), while `position.entry_price` is `bar_close + half_spread`. For BUY positions, `profit` here is always `half_spread` lower than the true mid-to-mid move. For SELL, `entry_price = bar_close - half_spread`, so `profit = entry_price - current_price` is also `half_spread` lower. This means trailing tiers activate slightly later than they would in a system that tracks profit from entry ask/bid to current bid/ask. With XAUUSD typical spread of $0.16, the shift is $0.08 — enough to delay BE tier activation by ~27% of its $0.30 threshold.

## Strong-momentum gate doesn't scale with ATR

```python
strong_threshold = Config.TICK_MOMENTUM_THRESHOLD * Config.STRONG_MOMENTUM_MULT
return move >= strong_threshold
```

`TICK_MOMENTUM_THRESHOLD` is $0.30, yielding `strong_threshold` = $0.45. This is a fixed dollar gate regardless of current volatility. In low-ATR regimes (ATR < $0.50), an 8-bar move of $0.45 is nearly the entire ATR and will rarely trigger, effectively capping positions at 1. In high-ATR sessions (ATR > $2), $0.45 triggers trivially, allowing rapid scaling to 5 positions. Consider applying the same ATR-ratio scaling that `_compute_threshold` uses.
